# 💫 About Me:
Hi there! 🚀 I’m a student and an embedded systems enthusiast who isn’t afraid to get my hands dirty with a soldering iron or spend all night staring at an oscilloscope. I’m an active member of my university’s solar car research circle, and in my free time, I build my own projects with a focus on reliability and solid engineering craft. 🛠️<br><br>What I’m hacking on right now:<br><br>Universal Bootloader (UB) for 15+ MCUs 🛰️ – My biggest project for the solar car team. I’m building a programming interface for a whole fleet of STM32s. Currently, I’m tackling the hardware layer: designing 4-layer PCBs and implementing communication via USB and RS-485. The goal is to deploy Azure RTOS (USBX) to keep everything stable and fast.<br><br>BMS Master 🔋 – The "brain" of the battery management system for our solar vehicle. I handle the cell monitoring logic and CAN bus communication in an environment full of electromagnetic noise.<br><br>PCBCells & ADC Driver 🧬 – My personal sandbox for mastering professional firmware. I’m writing an ADC driver from scratch for STM32 F1 and F3 series, strictly following TDD. I run all my unit tests (CUnit) on-target because I believe code has to work on the silicon, not just in a simulation.<br><br>My current learning path:<br><br>I’m currently transitioning from bare-metal programming to FreeRTOS 🔄. I also have big plans to dive into Cyber-security for embedded systems so I can eventually use TrustZone to secure my H5-based projects. On a daily basis, I use C/C++ for tooling, while designing PCBs with a focus on signal integrity—making sure every differential pair has its place (and a solid ground plane beneath it) 📐.<br><br>"Building systems resilient to bugs and noise, because the devil is in the details... and the ground loops." ⚡
